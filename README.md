## 我是怎么想到这个话题的

下午兴起，到浦江郊野公园去散步。边散步，边感受着周围的环境。有树，有草，有鸟鸣。好久没有这样静谧的体验了。渐渐，让我想起了在家乡和父亲在一起的时光，甚至还能想起父亲的音容，忽然就感伤起来了。于是，我放任自己的思绪奔走，沉浸在记忆中的那时、那刻，仿佛自己就在那里，就那样让回忆慢慢发生，那个场景就真的重现在了自己的周围，自己走到哪儿了都忘记了，就让自己的双脚随意走着。
一段思绪结束，回到了现实当中，我还在公园的小路上走着，周围稀稀疏疏地有几个爸爸妈妈带着他们的孩子在公园散步。小孩子稚嫩的声音萦绕过耳旁，飘荡到了远处的树林里去了。

我向着树林深处望去，隐约能望穿，却无法看清对面的任何光景。但忽然却让我陷入一个思考：到底记忆是什么？为什么我们会有记忆？为什么回忆起过去的故事总有积分忧伤？为什么回到故乡后又想尽快逃离那里？

我想找到答案。

## 到底记忆是什么

记忆是什么？回忆是什么？

> 记忆，是大脑对客观事物的信息进行编码、储存和提取的认知过程。也指存储信息的结构及其内容。包括识记、保持、回忆和再认。[百度百科](https://baike.baidu.com/item/%E8%AE%B0%E5%BF%86/34494)

这是一段从“百度百科”上搜索的关于记忆的描述。

在思考怎么定义记忆的时候，我也想到过“记忆”是某种类型的“数据”，放在我们的大脑里边，当我们需要的时候，像使用电脑的时候读取出来就可以了。但我本能地觉得这是一个特别符合直觉的“定义”，一定有什么地方“不太对”。但使用普通的解释，是无法明确它的，因为它无法被证明。所以，我想换一种方式来阐述，从而可以知道到底什么是记忆。

最近（其实已经不近了）AI已经差不多融入到了我的生活当中，至少在工作当中，使用它仿佛呼吸一般自然。忽然想到我的一个好友说过关于AI是如何工作的，他说“文学、艺术在AI中是无法验证真伪的。要想验证AI是否真的正确工作，就只能通过让AI进行可验证的作业才可以验证其是否真正的工作。”于是，我想尝试如何使用数学的方式来验证什么是记忆。

作为一个程序员，我一直在尝试的一个工作方式是不断地和AI进行交互，从而可以得到最终想要的结果。之所以有这样的一个过程，是因为目前我对AI的使用方式还停留在非常初级的阶段，尚未进行深度探索。和另外一个好友聊天时，他提到AI本身是有一个很大规模的矩阵进行不断运算得到的结果，它的本质就是不断进行计算并通过反馈回路重新进行计算，从而得到最符合预期的最大概率组合。那么，我们的记忆本身是不是一样是这样的进程呢？

于是我想大胆的做一个假设：我们的记忆本身也是一堆的不断计算的向量，涌动在我们的大脑里边。我们的记忆的过程，就是不断为这个向量增加维度、裁剪、汇总的过程？

## 记忆是怎么形成的

要说记忆是怎么处理事情的，我从一个可重复、可验证的我自身的一个例子开始：我是如何熟悉、处理一段很长的程序代码的。

当我接手到一段复杂的代码的时候，我会尝试从头到尾从“程序入口”开始一步步开始跳转、深入，直到把这段代码全部熟悉结束。这样的一个过程，它的本质，就是一个在构建记忆的过程。也就是说：我们思想里边原来没有这样的程序的概念，当阅读、分析完成后，我们就会有这样的一段关于这段代码的逻辑概念，它是模糊的、不精确的，同时，我们会给它提供了一个清晰的定义：这段程序主要做了什么。这时候，你可以尝试回忆一下自己学习一个新技能的过程，是不是类似这样的过程。

这时候我们没有“背诵”整段的代码，而是去分析、理解了它的结构、方法、计算、抽象、规则等等。有了这样的一个总结后，当你在项目的其他地方需要使用“类似”的功能、或者要使用这段的时候，你就可以迅速“想起”这段代码，然后可以迅速打开这段代码的文件，重新阅读它的关键部分，并检查它是否合适、如何使用等等。也就是说：

我们构造了这段记忆，它是这段代码的一个总结。

故事到这里还没有结束。

当阅读代码时，会不断地在不同的方法、接口之间跳转。导致的直接结果就是很容易读了前边忘了后边，读了后边又忘记前边的问题。这时候，同样的事还会发生：我们阅读一个方法，就会“哦！这个方法做了xxxx！”

也就是说，阅读整段代码的过程，是构造了一系列的“总结”。这些“总结”通过再次“汇总”，得到一个更大的“总结”，最终我们得到了这段代码的记忆。

所以，我们的记忆本身就是内核、记忆统一到一起的整体，它的向量在不断地增长、不间断运行的整体。

## 为什么做熟悉的事情会非常舒适

这有可能就是通常所讲的“舒适区”。

当我们去“记住”某些东西的时候，它的本质，就是一次次的汇总，拿到新的“总结”的过程。这个过程必定是一个相当耗能的过程，不然小孩子也不会那么逃避学习了。这个耗能的过程具体有多少还不清楚，肯定有某种方式可以测量出来，现在或者未来。

既然耗能，就是一件对大脑来讲很累的事情。大脑的其中一个作用就是要用最少的能量来完成它。其中一个就是逃避。如果避无可避，那就只能通过完整地掌握它，将它“编程”到大脑中，成为思维的一部分，从而可以在有需要的时候，以最小的代价获取到结果，而不是重新走一遍完整的过程去分析、计算、汇总得到一个结果。也就是说，我们的大脑是一个超级精密的懒人系统，它无时无刻在寻找快捷方式来降低对能量的消耗。通过这样的方式，就可以很好的理解我们掌握的技能，实际上就是“编程”到我们大脑里边的“熟悉”的快捷方式，当使用它们时，我们的大脑只要躺在椅子上用手指牵一下就拿到葡萄了。至于大脑为什么这样节省能量，有可能是氧气对身体的两面性吧。

我们将越多的快捷方式放在大脑中，那么思维的表现就是越“敏捷”，毕竟不需要重新计算了嘛。就类似于让你算22的平方，16的平方，有不少人是信手拈来，而同样有不少人需要打开计算器，或者用草稿去验算。

所以，如果能知道自己在什么岗位、需要什么技能、并熟练掌握后，就可以为自己量身打造舒适区。当然，除了一些乱折腾的组织。

## 回忆的时候发生了什么

很多人觉得回忆的时候，就是把过去拿出来重新播放一遍。我自己品味了一下自己的记忆，发现它有时候不太以这样的方式发生，它更像是“将记忆的总结重新放到了舒适区”，甚至对记忆本身进行了删减。因为我们无法完整地、精确地重述这段记忆，它就变成了一次粗略的提取，并让大脑分泌一些激素进行奖赏，从而让这样的记忆更可靠地存在于舒适区，避免它被丢失，毕竟它是大脑曾经消耗不少能量“编程”的结果，就这样丢失是非常不划算地一件事情。


记忆有的是和一个词语绑在一起，有的是和一个声音绑在一起，有的是和一个画面绑在一起，有的是和一个气味绑在一起，等等。也就是说，我们的意义的“总结”，不仅仅是一个总结，它还在形成的过程中，还被大脑添加了快捷标签，当特定的标签被提起时，对应的记忆的最粗的总结就会被拉到舒适区，然后才能通过一步步的将关联的“总结”提取出来，得到更多这块记忆的内容。当回忆的时候，我们的记忆大多是被牵着走的，而不是自己可以主动挑选、跳过的。直到整个回忆结束，我们才能重新总结我们的这段记忆，并记录到舒适区。就类似于想起一个歌曲的几个音调，但是就是想不起来这个歌曲的名字是什么，随着这首歌曲的音调的持续，歌曲的名字兴许就会出现在曲调中，最后“哦~~，我知道它的名字了！”然后迅速打开手机，搜索这首音乐播放起来，慢慢的回忆随着每个乐符跳跃在脑海中。

那么，我们怎么才能“回忆”这段记忆呢？它的随机性太高了。幸运的是，在AI当中，它可以将精确的记忆和记忆的总结一同存储起来，它可以精确地回忆每个字符。

## 为什么会急着逃离故乡

先不着急说逃离故乡的事情，先说一下“到底是谁动了我的代码！”或者“谁拿了我的笔！”

在程序员的世界，最可怕的事情莫过于另一个工程师“魔改”了我的代码，结果当我重新看这段程序的时候，我的大脑崩溃了，它进入了“panic”模式。因为当拿着辛辛苦苦构建、编程的舒适区的记忆回来的时候，发生了可怕的事情：怎么和大脑里边的记忆不一样了？！不一样的程度越大，大脑的不舒适就越强烈，因为它能够重用的“总结”越少，就不得不重新走一遍构建记忆的过程，重新消耗数不尽的能量来完成总结，并放入舒适区。同时，大脑会释放让人不舒适的激素进行惩罚，因为这些让本来进入另一种舒适模式时释放的另一种激素形成了相反的结果。代码越熟悉（比如：亲妈级别的代码），被重构的越彻底，崩溃的也越彻底。

那为什么我们要急着逃离故乡呢？如果是我，我怕！上一次回故乡也是几年前的事情了，体验极不舒适。因为差点找不到回去的大街，街坊的熟悉的房子几乎都不在了，除了门口还坐着几个苍老的长辈外，大街上几乎找不到熟悉的影子。儿时的破电线杆不见了，邻居的黑色大铁门没有了，对门的一层平房变成了三层小洋楼，只有自己家的门、房、树还有些过去的记忆。推开大门，满眼都是可没过脚踝的落叶，墙头上的草都长了老高，门窗上多么厚得浮沉，以前一开门欢迎我得大黄狗也没了，更没有了爸爸妈妈在厨房做饭、在客厅看电视的身影。风都懒得吹一下玻璃，生怕打破那一刻的沉寂。慢慢的都是记忆啊，那些独属于我的记忆，现在全都匹配不上了。我慌了，我的大脑崩溃了，它找不到它的舒适区了，这里全都是最熟悉的陌生。如果留下，就得重新从0开始，重新认识这个“家”，但我怕，我怕那些原来的记忆被“覆盖”了，找不到了。

所以，当我们处理全新的东西，从0开始学习的时候，我们的大脑准备好了冒险，它没有太多的假设，每一个都是新的，这是挑战。而当回到故乡时，这是一种冲撞，迫使大脑迅速进入防御姿态。本质上就是需要重新认识、重新构建记忆的过程。一直在老家生活的那些人不会有这样的感触，因为他们的记忆属于那里，属于那个舒适区。

## 为什么我们会手生

大脑在舒适区的处理速度是极其快速的，它可以迅速拿到声音、画面、气味、环境等等信息。但是，哪怕是维持舒适区，也是一件相对耗能的过程。可能维持的舒适区越大，维持它的代价也就越大，不然我们也不会在离开一个舒适区，进入下一个舒适区后，忽然跳回原来的舒适区也需要一小段的适应期。

也就是说，舒适区是有大小限制的。虽然目前还无法检测舒适区的大小，但它必定有一定的限制。就类似于AI处理的上下文有长度限制，当上下文达到极限时，就会触发压缩（compact），从而让整个计算过程都在可掌握的范围内。我们可以是超人，拥有超级舒适区，但是电脑有它的上限，至少，内存不免费。

## 为什么记忆没有时间概念

说起记忆有没有时间概念，我想说：可能、大概没有吧。

先举一个例子：回忆上周发生的事情，和回忆10年前发生的事情，基本是没有差别的。它们都是“稍加思考”就拿到了结果。甚至相反，10年前的还有可能更清晰。对比之下，你必须要靠特定的、客观的标准进行衡量到底是什么时候。就比如：那时我上几年级，或者那时候那棵树才多高，或者距离奥运会多久之类的，来给它一个参考。

如果记忆没有时间概念，也就意味着我们的大脑只能处理“现在”。所有的记忆都“活”在我们的脑组织当中，它需要7*24小时不间断运行，才能保证你过去构建的“知识”、“认识”、“见解”等等这些，都是需要活在脑中，才能在有不同的输入后，给出“最符合意图”的输出，或者“最节能”的答案。它永远活在“现在”。

如果没有时间的概念，我们怎么知道现在是什么时候呢？其中一种可能叫做“思维对齐”。就是我们的大脑学会了“现在”的定义，以及“日期”的定义，所以就将它们这些“超级快捷方式”放在了“舒适区”，每当有需要的时候，可以随时发起“思维对齐”，保证“现在”和现在对齐，从而保证在处理任何事情的时候，和其他人、事可以对应。如果一堆人对“现在”无法对齐，想想看会出现什么情况？比如：今天是周一，还是周二，还是周三？真是“今夕是何年”呀！

所以，记忆是活在“现在”，并且只对“现在”负责。

## 为什么会遗忘

我们的大脑的“舒适区”有很明确的上限（虽然不知道到底是多少），当我们急着“活”在“现在”时，舒适区的大小很快装不下那么多的内容，就会快速进行总结、汇总、压缩，以腾出来更多的空间、能量来处理“现在”。那么，有可能某一大片的记忆，会被混淆在一起，打上了不知名的标签继续“活”在组织当中。如果不再遇到类似的或者相同的标记，这段记忆就迷失在回忆里，直到某天，一颗石子，一片树叶，甚至一个喷嚏，就能让你重新想起。

## 如何用数学的方式来认识记忆

* 它永远是活的
* 它只活在现在
* 它只在舒适区处理
* 它的舒适区有长度
* 它在不断地积累
* 它在不断地压缩
* 它在不断地总结
